A Brief Description

In this Project we have designed a space shooting game similar to the ones found in the earlier Nokia phones, where there is a spaceship, controlled by the player. The ship needs to destroy the approaching enemies. In case of a collision with the enemy, it loses some health. After loss of a certain amount of health, the game overs.

LED matrix is used to show the game board, and LCD display module is used to show other information. Gyroscope is used to move the ship. Two ATmega32 microprocessors powers the whole system.





Required Equipments

  • ATmega16 microcontroller

  • Gyroscope

  • Two 8x8 LED Matrix Module

  • LCD Display Module

  • 4 to 16 Decoders

  • And some more things to interface them:

    • 5V Fixed 1A Positive Power Supply Regulator

    • Registors

    • Capacitors





How it works?

If you are also a 90's kid like us, you must remember growing up playing those awesome games on your parents first phones! Ah, how great those days were! We have created this game to remind you those lost golden days. We have made a space shooter game where you, the player, run a spaceship. You can control which direction you are going. There are enemies/objects approaching you (or you approaching them)! A collision with any of those objects is very deadly for your spaceship. The easiest way to continue moving is to destroy these objects as you find them. You can shoot at a certain interval to destroy them. Destroying them increases your score in intergalactic tourist rank, because, guess what, you are making future journeys easier for others! However, you can't always destroy those enemies. And in that case you need to avoid them, as a collision may be very deadly for spaceship. You may somehow survive one, two or three, but the best ship available in the universe right now (yes, that's yours!) survive five such collisions :'(

Algorithm



       1.Start
              2.GSM module's receiver turned on
              3.GPS receiver remains turned off
                     4.GSM waits for an SMS request
                            5.If receives a request
                                   6.If the request is valid
                                          7.Turn off the GSM receiver
                                          8.Turn on GPS receiver
                                          9.Get GPS data
                                          10.Turn off GPS receiver
                                          11.Turn on GSM receiver
                                          12.Parse data and get the Location
                                          13.Send an SMS including position information
                                   14.Else go to (4)
                            15.Else go to (4)
       16.End

Problems Faced

  1. After we had finished writing the codes for our project we designed the circuit in Proteus simulator. Everything was working as expected. However, when we implemented the same circuit in hardware, it was not working. We tried to find out reason, but didn't find any. Finally a tweak in the codebase let us solve the problem.

  2. After we had connected both the bicolor LED matrices, they started to behave randomly due to voltage fluctuation, caused by wire connections getting loose after a movement.

  3. Initially we had written the whole codebase in C++. But the library that we used to interface our gyroscope with ATmega32 module was written in C. Compiling both using the same compiler created some compatibility issues, which was fixed by compiling the C code with a C compiler and using a different ATmega32 for that.
